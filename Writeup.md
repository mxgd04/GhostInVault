# Writeup: Ghost in the Vault

**Category:** Reverse Engineering  
**Difficulty:** Easy  
**Event:** 47CON CTF  

## Initial Analysis
Loading the binary into tools like Ghidra or IDA Pro reveals that all human-readable strings have been removed. Furthermore, the main execution flow is packed into a massive `while` loop governed by a central `switch` statement. This is a classic implementation of **Control Flow Flattening**.

Additionally, all text strings are computed at runtime using a decryption function (`xs`) that applies a rotating XOR key (`0xA7 ^ deltas`).

## Evasion of Defenses (Anti-Debug)
Attempting to run the program with GDB or `ltrace` causes it to terminate immediately. Analyzing the initial execution flow, we find a function called during every iteration of the dispatcher. This function implements 5 layers of defense:

1. `ptrace(PTRACE_TRACEME)`
2. Scanning `/proc/self/status` for `TracerPid`
3. Timing checks using `rdtsc`
4. Checking for the `LD_PRELOAD` environment variable
5. Memory scanning for `0xCC` bytes (Software Breakpoints)

## Key Derivation and Real Cryptography
The final routine extracts a 27-byte data block (`store[]`) and processes it with:

1. **XTEA (Extended TEA):** Decrypts the block.
2. **S-BOX XOR:** Applies a byte-by-byte XOR using a custom substitution table.

This decryption requires a 4-byte (32-bit) **Master Key**. This key is dynamically generated by mixing the file's Checksum, a hardcoded compiled seed (`0x52EC`), the system's UID, and the Hostname, processing them through 3 rounds of the **ChaCha** algorithm (Quarter-round).

## The Exploit (Cryptographic Brute-Force)
Since we do not know the administrator's original hostname (`debian`) or UID (`1000`), the binary will not yield the correct flag on our machine. However, there is an intentional flaw in the defensive design: **the master key is only 32 bits long**.

Instead of guessing random environments, we can extract the encrypted `store[]` block and the `sbox` array from the binary, and write a script to brute-force the keyspace. A more elegant and much faster approach is a dictionary attack targeting common Linux hostnames and UIDs.

import struct

def attempt_decrypt(hostname, uid):

    if flag.startswith(b"47CON{"):
        print(f"[+] WIN! Host: {hostname}, UID: {uid}")
        print(f"[+] FLAG: {flag.decode(errors='ignore')}")

# Dictionary of typical CTF/Linux environments
hosts = ["kali", "ubuntu", "debian", "arch", "localhost"]
uids = [0, 1000, 1001]

for h in hosts:
    for u in uids:
        if attempt_decrypt(h, u):
            break
